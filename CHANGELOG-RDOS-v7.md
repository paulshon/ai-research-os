# CHANGELOG — AI Research OS + RDOS v7

v6 기반. 플랜 교차 접근·미션 연동 승급·문의 연동·가입 구조(연구자 마법사)·
7단계 인증·사용자 흐름을 구현. 버전 7.0.0.

## 1. 플랜 교차 접근 + 미션 완료 기반 승급
- **연구자 → RDOS 접근**: 승인된 연구자(researcher_enrollment approved/active)는
  RDOS 에도 로그인 가능하도록 `isRdosAllowed` 확장(플랜 교차 접근).
- **RDOS → 연구자 승급(미션 게이트)**: 연구준비자는 프로그램 미션을 모두 완료한 경우에만
  연구자 플랜으로 승급. 서버에서 미션 완료를 재검증한 뒤에만 승급 처리.
- **관리자 연동**: RDOS 관리자에 `미션 현황·승급` 탭 신설 — 회원별 미션 완료 수·진행률·
  승급 자격을 커널(`getLearnerState`)로 도출해 표시하고, 자격 충족 시에만 `연구자 승급` 실행.
  - API: `/api/admin/rdos/missions`(현황), `/api/admin/rdos/upgrade`(미션 게이트 승급).
- 신규 테이블 `researcher_enrollment`(마이그레이션 `0013`).

## 2. 랜딩 문의하기 → 관리자 페이지 연동
- 문의 폼이 실제로 값 수집 후 `/api/contact` 로 전송(이전엔 전송 없이 완료 표시만).
- `/api/contact` 가 **contact_inquiries 테이블에 저장**(POST), 관리자만 조회/상태변경(GET/PATCH).
- 관리자 페이지(`/admin`)에 **📨 랜딩페이지 문의** 섹션 추가 — 신규/읽음/완료 상태 관리.
- 마이그레이션 `0012_contact_inquiries.sql`. 랜딩 푸터 `문의` → `/contact` 연결(기존).

## 3. 회원가입 구조
- **연구준비자(RDOS, 사양 A)**: 이름·이메일·소속대학·학과(필수), 석사과정여부(선택),
  연구관심분야(필수). 가입조건=석사과정 재학(학번 기입으로 대체). 기존 마법사 보정.
- **연구자(Researcher, 사양 B)**: 신규 7단계 마법사(`/researcher-signup`).
  이름·이메일·소속기관·연구분야(필수), ORCID·Google Scholar(선택),
  대표논문링크 1·2·석사/학술논문 링크(필수). 가입조건=**학술/석사 논문 2편 이상 + 링크 검증**
  (클라이언트 + 서버 양쪽 검증). API `/api/researcher/enroll`.

## 4. 인증 프로세스 7단계
- 두 마법사 모두 동일 7단계: 플랜선택 → 가입정보 → 논문/학위정보 → 이메일인증 →
  로그인(Google/GitHub/Email) → 관리자승인 → 서비스진입.

## 5. 사용자 흐름
- 네비게이션에 `연구준비자 플랜`(→/rdos-signup)·`연구자 플랜`(→/researcher-signup) 진입 추가.
- 흐름: 랜딩 → 플랜 선택 → 회원가입(7단계) → 인증 → 관리자 승인 →
  RDOS(연구준비자) / AI-Research-OS(연구자), 연구자는 RDOS 도 교차 이용.

## 검증
- `apps/web` 전체 `tsc --noEmit` clean.
- 신규 API/마이그레이션/마법사/관리자 탭 배선 확인. 이메일 평문 미노출 유지(base64).

## 배포 시 필수
- 신규 마이그레이션 적용: `supabase db push` (0012 문의, 0013 연구자 등록).
- 연구자/대시보드 정식 진입 게이팅(승인 전 차단)은 운영 정책에 따라 선택 적용 가능
  (현재는 enrollment 기록·교차접근·미션 승급 연동까지 구현, 대시보드는 기존 접근 유지).
