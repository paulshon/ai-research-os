## 3. 5개 메뉴 흐름 데모 — 실제 메커니즘 한글 재현 (데스크탑/태블릿/모바일 공통)

각 메뉴 페이지의 실제 동작을 분석해 데모를 실제 단계·한글 콘텐츠로 재작성:
- **문헌연구**: 논문 검색 → 추천 논문(저자·연도·피인용) → 인용 담기 → 연구 갭 분석 (실제 4단계).
- **논문구조엔진**: 논문 유형(양적/질적/혼합/실험) → 서론·이론배경·연구방법·결과·논의·결론 6챕터 생성 →
  선택 챕터의 세부 단락(연구 설계·표본·도구·분석 방법) 펼침.
- **논문크리틱**: 업로드 → AI 분석 → 원문 번호 배지 + 패널 카드 + 번호↔카드 연동 강조.
- **참고문헌정리**: DOI 조회 → Crossref→OpenAlex→Semantic Scholar 다중 출처 → 실제 7개 인용 스타일
  (APA7/MLA9/Chicago/IEEE/Vancouver/Harvard/Nature) 순환 렌더.
- **문장라이브러리**: 논문 유형·챕터 선택 → AI 학술 문장 패턴(맥락 제시/연구 공백/연구 목적/학술 기여)
  실제 예시 문장과 함께 생성.
- 캡션·콘텐츠 전부 한글화(ko), en/zh 캡션도 동기화. 자동 순환 + 탭 클릭 + 진행 바 유지.

## 4. 모바일 변환텍스트 크리틱 생성 — 재수정 (데스크탑 동작 분석 기반)

데스크탑은 `onMouseUp` → `window.getSelection()` → floating 팝업으로 정상 동작하나,
모바일은 (a) touchend 시점 selection 불안정, (b) floating 팝업이 좌표 기준이라 화면 밖/가림으로
보이지 않던 것이 근본 원인.

- **전역 `selectionchange` 리스너**(디바운스)로 선택 확정을 안정적으로 감지하도록 재설계.
  선택 영역이 `data-selectable` 컨테이너(변환텍스트/텍스트모드/원문 재현) 안인지 확인하고
  `data-page`로 페이지 매핑.
- **모바일 전용 하단 액션 시트**로 팝업을 분리 — 화면 하단 고정이라 좌표 어긋남/가림 없이 항상 노출,
  큰 터치 타겟(유형 선택 + 크리틱 생성). 데스크탑은 기존 floating(화면 경계 클램프 추가) 유지.
- `isMobile`(<1024px) 감지로 분기. floating 팝업의 `t` 변수명 충돌(i18n `t`)도 `tp`로 정리.
- 결과: 모바일에서 변환텍스트의 단어를 드래그하면 하단 시트가 떠 크리틱이 생성되고 번호·패널과 연동됨.
