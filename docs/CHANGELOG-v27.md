# CHANGELOG v27 — 테마 시스템 재설계 (3가지 다크 테마)

## 배경
v26의 테마는 Light/Gray/Dark 혼합 방식이었는데, 라이트 모드에서 사이드바는
어둡게 남고 본문만 흰색으로 바뀌는 등 **반쪽짜리로 깨지는 문제**가 있었습니다.
원인은 앱 전반이 수백 개의 하드코딩 색상(`bg-[#13161e]`, `text-white/40` 등)을
쓰는데, 라이트 전환 시 그 모든 흰색 텍스트·어두운 배경을 일일이 반전시켜야 했고
일부만 처리되어 일관성이 깨진 것이었습니다.

## 변경 — 사용자 제공 2색 + 검정으로 3가지 다크 테마
첨부된 두 샘플 색상과 검정색을 기반으로, **모두 어두운 계열의 3개 테마**로
재설계했습니다. 세 테마 모두 다크 계열이라 텍스트는 밝게 유지되므로, 배경 표면만
바꾸면 되어 깨짐 없이 일관됩니다.

| 테마 | 색상 | 비고 |
|------|------|------|
| **미드나잇 (midnight)** | `#0d0212` | 퍼플-블랙 (샘플 1) — 기본값 |
| **차콜 (charcoal)** | `#1f1f1d` | 웜 차콜 (샘플 2) |
| **블랙 (black)** | `#000000` | 순수 블랙 |

## 구현
- `components/theme/theme-provider.tsx` — 테마 ID를 midnight/charcoal/black으로 교체,
  localStorage 영속화, 기본값 midnight, 깜빡임 방지 인라인 스크립트 갱신
- `components/theme/theme-switcher.tsx` — **색상 스와치 미리보기**가 있는 세그먼트
  스위처(설정용) + 상단바용 원형 스와치 토글(순환)
- `app/globals.css` — 테마 시스템 전면 교체:
  · 3개 테마의 배경 표면 계단(`--bg` ~ `--surface-deep`) 토큰 정의
  · 하드코딩 배경 hex(`#0d0f14`, `#13161e`, `#1a1e2a`, `#222637`, `#1e2230`,
    `#0f1117`, `#0f1118`, `#0a0c10`)를 **모든 data-theme에서 토큰으로 재매핑**
    → 사이드바·본문·카드·패널이 한꺼번에 일관되게 전환
  · v26의 light/gray 반전 규칙 전부 제거 (깨짐 원인 제거)
- `app/layout.tsx` — 기본 `data-theme="midnight"`
- `lib/i18n/locales/*` — 테마 설명 문구를 3개 다크 테마에 맞게 갱신(ko/en/zh)

## 의도적으로 유지한 부분
- 강조색(`#6c8cff`, `#4a6cf7`, `#ff7066` 등)과 밝은 텍스트는 3개 테마 공통 유지
- `document-reproduction.tsx`의 흰 종이 미리보기(원문 재현 "종이 같은 표면")는
  인쇄물처럼 의도적으로 밝게 유지 — 테마 영향 없음
