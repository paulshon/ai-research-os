## 1. 히어로 한글 헤드라인 깨짐 버그 수정 (첨부 1번 이미지)

증상: 히어로 첫 줄 한글이 "학VV 04 OuyWa iye K9T Zps PXX"처럼 라틴 글리프로 치환된 채
원문으로 복구되지 않고 멈춤.

원인: `ScrambleText`가 (a) 한글을 라틴 글리프로 셔플해 깨져 보였고,
(b) `started` 가드 + `text` 의존 `run` 콜백 + setTimeout 경쟁으로 리렌더/언어전환 시
프레임이 꼬여 원문으로 수렴하지 못함.

수정:
- 컴포넌트 재작성 — **한글 등 비라틴 문자는 셔플 대상에서 제외**(`isLatin`)하고 원문 유지.
  항상 원문으로 수렴하도록 `setInterval` 단일 타이머 + 엄격한 정리, `text` 변경 시 즉시
  원문 리셋 및 가드 해제.
- **히어로 거대 헤드라인에서는 스크램블을 아예 제거**하고 blur-reveal 모션으로 대체
  (대형 한글 텍스트의 깨짐 위험 원천 차단). 스크램블은 라틴 라벨/버튼에만 안전하게 사용.

## 2. DesignMe 스타일 라이트 리뉴얼 (랜딩 완전 리뉴얼)

첨부 분석 글 + 스크린샷 기준으로 다크 → **라이트 테마**로 전면 전환:
- **색상 시스템**: 배경 #FFFFFF, 텍스트 #07111F, 보조 그레이, 액센트 오렌지(#ff7a00) —
  95% 무채색 원칙.
- **타이포그래피 주도**: 중앙 정렬 초대형 산세리프 헤드라인(`clamp`로 최대 5.2rem,
  line-height 1.02, 자간 -0.02em). "텍스트가 곧 디자인".
- **Hero**: 상단 그래디언트 메시 글래스 오브젝트(오렌지/옐로/화이트 글로우, 회전 글래스 카드),
  소문자 모노 라벨 → 헤드라인 → 설명 → CTA 2개 → 연구 흐름 칩의 전형적 SaaS 히어로 구조.
- **Client/Source Logo Bar**: Swiss 그리드(세로 구분선) 위 모노크롬 소스 로고
  (Crossref·OpenAlex·Semantic Scholar·GROBID·ORCID·DOI), whileInView reveal.
- **Features**: 카드 대신 그리드 셀(1px 구분선) 에디토리얼 레이아웃, hover 시 미세 배경/리프트.
- **Architecture·Workflow·Pricing·CTA·Footer**: 전부 라이트(흰색/연회색 #fafbfc 교차 섹션,
  검정 CTA 버튼, Pricing 강조 플랜만 다크 반전), blur-reveal/stagger 모션 적용.
- **네비**: 플로팅 고정(76px), 라이트, 검정 워드마크 + 오렌지 OS, 실시간 모노 클럭(라이트 색).
- **모션 쇼케이스**(전체 메뉴·5개 메뉴 흐름): 섹션 헤더/좌측 번호 리스트는 라이트로,
  데모 프레임은 다크 디바이스 목업으로 유지(라이트 페이지 위 대비 — Linear/Vercel식).
  5개 메뉴 흐름의 실제 메커니즘·한글 콘텐츠(v36)는 그대로 유지.
