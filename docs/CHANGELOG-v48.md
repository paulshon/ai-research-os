## 1. 결론(Conclusion)이 2번 나오던 문제 해결 (그림 1~4)

- 원인: 제목 현지화 로직(`lib/i18n/research-labels.ts`)이 챕터 **번호**(CH.06·CH.07)를 모두
  `결론`으로 강제 매핑하고, 모든 유형의 제목을 번호 기준으로 덮어쓰고 있었음.
  → 7장 구조(개념분석 ca · 정책분석 pa · 실행연구 ar · 평가연구 er)에서 06·07이 둘 다 "결론"으로 표시됨.
- 수정: 번호 기반 매핑을 제거하고 **정규 섹션 제목 기반**으로 변경.
  - 한국어: 작성된 챕터 제목을 그대로 사용(정본) → 각 유형이 고유 구조·단일 결론으로 정확히 표시.
  - EN/ZH: 콘텐츠 오버라이드 우선, 없으면 정규 섹션만 번역하고 고유 제목은 유지.
- 결과: ca/pa/ar/er 모두 **결론 1개**로 정리(검증: 각 7장·결론 1).

## 2. 사례연구(⑯)·도구개발(⑰) 구조엔진 신설 (업로드 가이드 반영)

업로드된 두 가이드(사용자 제공)를 분석하여 기존 유형과 동일한 형식으로 구조 데이터를 추가했습니다.

- **사례 연구 (cs)** — Yin(2018)/Merriam & Tisdell 기반 **7장**:
  서론·연구배경 → 이론적 배경 및 연구 명제 → 사례 연구 설계 → 자료 수집 및 사례 기술 →
  사례 분석 및 발견 → 논의 → 결론. (명제, 패턴 매칭, 분석적 일반화, 두꺼운 기술 등 미시 구조 반영)
- **도구 개발 (id)** — DeVellis(2016)/Comrey & Lee(1992) 기반 **8장**:
  서론 → 이론적 배경 및 구인 정의 → 문항 생성 방법론 → 전문가 검토·예비연구 →
  연구1(문항분석·EFA) → 연구2(CFA·타당도) → 논의 → 결론. (CVI, PAF·사각회전, CFA 적합도,
  수렴·판별 타당도 등 미시 구조 반영)
- `lib/research-data.ts`에 `CHAPTERS_CS`/`CHAPTERS_ID` 추가, `getChapters`에 `cs`/`id` 연결.
  (이전에는 두 유형이 양적연구 구조로 잘못 대체되었음)
- 각 챕터는 기존과 동일하게 `desc/question/macro/micro/goodPatterns/badPatterns` 포함.

## 3. 논문작성 — 그림/표 수동(드래그) 크기 조절 수정 (그림 2)

- 증상: 프리셋(작게/보통/크게)은 되지만 코너 핸들 **드래그로 수동 조절이 안 됨**.
- 원인: v47의 `setPointerCapture` 방식이 리사이즈 중 React 리렌더와 충돌해 move 이벤트가 끊김.
- 수정: 드래그 시작 시 **`window`에 pointermove/pointerup 리스너**를 직접 부착하는 방식으로 변경
  (`components/editor/rich-text-editor.tsx`). 핸들 그랩 영역 확대 + `touch-action:none`.
  → 코너를 끌어 8~100% 범위로 부드럽게 수동 리사이즈 가능(이미지/표 공통, 모바일 포함).

## 4. 설정 페이지 레이아웃 정리 (그림 5)
