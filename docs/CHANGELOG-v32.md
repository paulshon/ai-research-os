## 1. 논문 크리틱 — 원문/변환텍스트 ↔ 크리틱 패널 양방향 연동 (모바일 포함)

**문제**: 모바일에서 논문 업로드 후 변환텍스트·원문에 크리틱 번호가 표시되지 않고,
오른쪽 크리틱 패널과의 연동이 동작하지 않았습니다.

**개선**
- PDF 모드의 **페이지 변환텍스트**에도 크리틱 번호 배지를 표시(기존에는 페이지 이미지에만 표시).
- 번호 ↔ 카드 **양방향 연동** 추가
  - 원문/변환텍스트의 번호를 탭 → 해당 크리틱 카드로 이동(모바일은 크리틱 페인 전환) + 강조
  - 패널 카드의 번호를 탭 → 원문 해당 위치로 이동(모바일은 뷰어 페인 전환) + 강조
- 활성 카드 하이라이트(링) 표시로 현재 선택 위치를 명확히 함.
- AI 자동 분석 결과의 페이지 매핑 강화: 공백 정규화 + 단계적 키 길이 매칭,
  실패 시 1페이지로 폴백하여 원문에 번호가 항상 노출되도록 보정.
- 사이드바 페이지 클릭 시 모바일에서도 뷰어 페인으로 전환되어 바로 이동.

## 2. 논문 크리틱 — 400페이지 대용량 PDF 원문 표시 (3개 버전 공통)

**문제**: 업로드는 되지만 원문(페이지 이미지)이 보이지 않고 변환텍스트만 표시.

**원인**: 기존 `renderAllPages`가 120페이지 초과 PDF의 페이지 이미지를 통째로 생략
(`IMAGE_PAGE_CAP = 120`)하고 텍스트만 추출했기 때문.

**개선 — 지연 렌더링(Lazy Rendering)으로 전환**
- 업로드 즉시 **모든 페이지의 텍스트만** 추출(크리틱 분석의 핵심 소스, 매우 가벼움).
- 살아있는 PDF 문서 핸들을 보존하고, 화면(뷰포트) 근처에 들어온 페이지만
  그 순간 캔버스로 렌더(`LazyPdfPage` + IntersectionObserver).
- 화면에서 멀어지면 이미지를 해제하여 메모리 상한을 유지.
- 결과적으로 **페이지 수와 무관하게(400p+) 원문이 항상 표시**되며 메모리 사용이 일정.
- 저장본 복원 시에도 원본 PDF 바이트로 문서 핸들을 다시 열어 원문을 재현
  (localStorage 용량 초과로 원본 바이트가 제거된 경우에만 텍스트 전용으로 우아하게 강등).

## 3. 랜딩 페이지 — 전체 메뉴 & 기능 + 논문분석 흐름 Motion (Framer Motion)

`framer-motion`을 랜딩 페이지에 탑재하고, v31의 전체 메뉴/기능을 분석하여 시각화했습니다.

- **All Menus & Features** 섹션: 연구 흐름 / AI 도구 / 공통 도구로 그룹화한
  전체 메뉴를 `whileInView` stagger + hover 마이크로 인터랙션으로 표현.
- **Paper Analysis Flow** 섹션: 요청하신 UX→Motion 시나리오를 자동 재생 데모로 구현
  - PDF 업로드(드롭존 pulse) ↓ fade-in
  - 분석 시작 ↓ progress + pulse
  - 분석 완료 ↓ 카드 등장(fade/scale)
  - 키워드 분석 ↓ stagger
  - 연구방법 분석 ↓ stagger
  - 참고문헌 분석 ↓ slide-up
  - `prefers-reduced-motion` 존중(모션 최소화 시 결과 즉시 표시)
- 히어로 텍스트·CTA·프레임워크 칩·기능 카드 그리드에 fade-in / stagger 적용.

**새 파일**: `apps/web/components/marketing/motion-showcase.tsx`
**i18n**: `landing.*`에 신규 키 추가(ko/en/zh).
