## 7. 논문 크리틱 — 스크롤 시 사이드바·패널 사라짐 해결 (그림 7)

증상: 논문을 불러와 페이지가 길어지면, 전체 페이지가 함께 스크롤되어 왼쪽 사이드바(전체 메뉴·
페이지 썸네일)와 오른쪽 크리틱 패널이 화면 밖으로 밀려 사라짐.

원인: 대시보드 `<main>`이 스크롤 컨테이너인데 크리틱 페이지 루트에 높이 제약이 없어
3-pane 전체가 문서 흐름으로 늘어나 통째로 스크롤됨(각 pane 내부 `overflow-y-auto`가 무력화).

수정(데스크탑 lg+ 한정, 모바일 3-pane 전환 동작은 그대로 유지):
- 크리틱 페이지 루트를 `lg:h-[calc(100dvh-2.5rem)] lg:overflow-hidden`으로 뷰포트 높이에 고정.
- 3-pane 래퍼·좌측 사이드바·중앙 뷰어·우측 크리틱 패널에 각각 `lg:h-full lg:overflow-hidden` 적용.
- 결과: 사이드바 헤더(업로드·논문 불러오기)와 크리틱 패널 헤더는 항상 고정되고, 썸네일 리스트·
  원문 뷰어·크리틱 카드 리스트가 **각자 독립적으로 내부 스크롤**되어 패널이 사라지지 않음.
