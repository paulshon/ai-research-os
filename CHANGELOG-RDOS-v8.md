# CHANGELOG — AI Research OS + RDOS v8

v7 기반. RDOS 학습 대폭 증설(모듈당 20문항)·RDOS 반응형·문의 견고화·Clerk 인증 흐름 수정. 8.0.0.

## 1. RDOS 학습 모듈 증설 + 모듈당 20문항 퀴즈 + 쉬운 설명
- 논문가이드 5-4 전체(8개 장 서술구조·탈락패턴 15·좋은문장 15) + 31개 용어 +
  양적 9/질적 7/혼합 4 방법론을 분석·적용.
- **학습 모듈 60개**(basics 8 · structure 10 · design 8 · method 8 · reading 7 ·
  apa 6 · writing 8 · tutor 5)로 증설.
- **모든 모듈에 20문항 이상 퀴즈** — 총 **1,200문항**(모듈 내 질문 중복 없음, 정답/보기 무결성 검증).
- 각 모듈 본문은 **쉬운 설명(쉽게 말하면) + 예시(예시로 이해하기) + 정확한 정의/구조**로 구성.
- 커널(`RDOS_MENU_MAP`)의 메뉴별 레슨 수를 실제 모듈 수와 동기화(미션 진행·승급 정확도).

## 2. RDOS 반응형(데스크탑·태블릿·모바일/iOS·Android)
- RDOS 셸을 AI-Research-OS 대시보드와 동일한 적응형 구조로 재작성:
  - 데스크탑(lg+) 풀 사이드바 · 태블릿(md~lg) 컴팩트 레일 ·
    모바일(<md) 상단 헤더 + 하단 내비 + 전체메뉴 시트 + 풀 사이드바 오버레이.
  - iOS 노치 대응(`env(safe-area-inset-top/bottom)`), `100dvh` 적용 →
    안드로이드/iOS 모바일 브라우저에서 안전 영역·스크롤 정상.

## 3. 랜딩 문의하기 → 관리자 연동 견고화
- 문의 저장을 **service role 키가 없어도 anon 키로 저장**되도록 폴백 + 마이그레이션
  `0012` 에 anon INSERT 정책 추가(공개 폼 제출 보장). 관리자(`/admin`)의 문의 섹션에서 조회.
- 서버 DB 미설정 시 가짜 성공 대신 명확한 안내(503) 반환.

## 4. Clerk 회원가입 인증번호 후 진행 멈춤 수정
- 원인: `<SignUp routing="path">`/`<SignIn>` 가 단일 페이지라 이메일 인증 등 하위 단계
  경로(`/signup/verify-email-address` 등)를 처리하지 못해 인증 화면에 멈춤.
- 수정: `signup`·`login` 을 **옵셔널 catch-all 라우트**(`[[...rest]]`)로 전환 →
  인증번호 입력 후 정상적으로 다음 단계·리다이렉트(`/onboarding` 등) 진행.
